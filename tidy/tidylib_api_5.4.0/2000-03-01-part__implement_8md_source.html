1 ## Implement **LibTidy** {#libtidy_03}
2 
3 Once you’ve built **LibTidy** following the README instructions for `CMake` in
4 **HTML Tidy’s** repository, you can get started using **LibTidy**. `CMake` will
5 have built both the console application and the library for you.
6 
7 Perhaps the easiest way to understand how to use **LibTidy** is to see a simple
8 program that implements it. Such a simple program follows in the next section,
9 and don't forget that you can also study `console/tidy.c`, too.
10 
11 Before we look at the code, it’s important to understand that API functions that
12 return an integer almost universally adhere to the following convention:
13 
14 0 == Success, good to go.
15 
16 1 == Warnings, but no errors. Check the error buffer or track error messages for details.
17 
18 2 == Errors (and maybe warnings). By default, Tidy will not produce output. You can force output with the
19  `TidyForceOutput` option. As with warnings, check the error buffer or
20  track error messages for details.
21 
22 <0 == Severe error. Usually the value equals `-errno`. See `errno.h`.
23 
24 
25 Also, by default, warning and error messages are sent to `stderr`.
26 You can redirect diagnostic output using either `tidySetErrorFile()`
27 or `tidySetErrorBuffer()`. See `tidy.h` for details.
