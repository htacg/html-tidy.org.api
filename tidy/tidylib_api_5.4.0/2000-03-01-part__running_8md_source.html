1 # Running Tidy {#tidy_console}
2 
3 ## Running Tidy in a Terminal (Console)
4 
5 This is the syntax for invoking **Tidy** from the command line:
6 
7 `tidy [[options] filename]`
8 
9 **Tidy** defaults to reading from standard input, so if you run **Tidy** without
10 specifying the `filename` argument, it will just sit there waiting for input
11 to read.
12 
13 **Tidy** defaults to writing to standard output. So you can pipe output
14 from **Tidy** to other programs, as well as pipe output from other programs to
15 **Tidy**. You can page through the output from **Tidy** by piping it to a
16 pager, e.g.:
17 
18 `tidy file.html | less`
19 
20 To have **Tidy** write its output to a file instead, either use the
21 
22 `-o filename` or `-output filename`
23 
24 option, or redirect standard output to the file. For example:
25 
26 ~~~
27 tidy -o output.html index.html
28 tidy index.html > output.html
29 ~~~
30 
31 Both of those run **Tidy** on the file **index.html** and write the
32 output to the file **output.html**, while writing any error messages to
33 standard error.
34 
35 **Tidy** defaults to writing its error messages to standard error (that is, to
36 the console where youâ€™re running **Tidy**). To page through the error messages
37 along with the output, redirect standard error to standard output, and pipe
38 it to your pager:
39 
40 `tidy index.html 2>&1 | less`
41 
42 To have **Tidy** write the errors to a file instead, either use the
43 
44 `-f filename` or `-file filename`
45 
46 option, or redirect standard error to a file:
47 
48 ~~~
49 tidy -o output.html -f errs.txt index.html
50 tidy index.html > output.html 2> errs.txt
51 ~~~
52 
53 Both of those run **Tidy** on the file **index.html** and write the
54 output to the file **output.html**, while writing any error messages to
55 the file **errs.txt**.
56 
57 Writing the error messages to a file is especially useful if the file you
58 are checking has many errors; reading them from a file instead of the
59 console or pager can make it easier to review them.
60 
61 You can use the or `-m` or `-modify` option to modify (in-place) the contents
62 of the input file you are checking; that is, to overwrite those contents with
63 the output from **Tidy**. For example:
64 
65 `tidy -f errs.txt -m index.html`
66 
67 That runs **Tidy** on the file **index.html**, modifying it in place
68 and writing the error messages to the file **errs.txt**.
69 
70 **Caution:** If you use the `-m` option, you should first ensure that you have a
71 backup copy of your file.

74 ## Running Tidy in Scripts
75 
76 If you want to run **Tidy** from a Perl, bash, or other scripting language
77 you may find it of value to inspect the result returned by **Tidy**
78 when it exits: `0` if everything is fine, `1` if there were warnings
79 and `2` if there were errors. This is an example using Perl:
80 
81 ~~~
82 if (close(TIDY) == 0) {
83  my $exitcode = $? &gt;&gt; 8;
84  if ($exitcode == 1) {
85  printf STDERR "tidy issued warning messages\n";
86  } elsif ($exitcode == 2) {
87  printf STDERR "tidy issued error messages\n";
88  } else {
89  die "tidy exited with code: $exitcode\n";
90  }
91 } else {
92  printf STDERR "tidy detected no errors\n";
93 }
94 ~~~
